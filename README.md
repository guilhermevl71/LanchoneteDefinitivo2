A arquitetura adotada no sistema foi baseada no padrão de aplicação em camadas, separando as responsabilidades entre frontend, backend e banco de dados, com o objetivo de facilitar a organização, manutenção e escalabilidade do projeto. O sistema foi desenvolvido utilizando ASP.NET Core Web API no backend, responsável por processar as regras de negócio, autenticação de usuários, controle de acesso com JWT e comunicação com o banco de dados MySQL através do Entity Framework Core. Já o frontend foi desenvolvido utilizando HTML, CSS, JavaScript e Bootstrap, sendo responsável pela interface visual e interação do usuário com o sistema.

A arquitetura da aplicação foi organizada em diferentes componentes. O módulo de Models contém as entidades principais do sistema, como Usuário, Produto, Pedido e ItemPedido, representando as tabelas do banco de dados. O módulo Data é responsável pela configuração do contexto do banco de dados através da classe LanchoneteContext, além dos DTOs utilizados para transferência de dados entre cliente e servidor. O módulo Controllers concentra os endpoints da API, responsáveis pelas operações de cadastro, login, gerenciamento de produtos e carrinho. A autenticação foi implementada utilizando JWT (JSON Web Token), permitindo proteger rotas administrativas através de roles, diferenciando usuários comuns de administradores.

A FIGURA 1 representa o Diagrama Entidade-Relacionamento (DER) do sistema, demonstrando a estrutura das tabelas e seus relacionamentos, como a associação entre pedidos, produtos e usuários. Já a FIGURA 2 representa o Diagrama de Caso de Uso, apresentando as principais funcionalidades disponíveis para clientes e administradores, incluindo login, cadastro, visualização do cardápio, gerenciamento de produtos e utilização do carrinho de compras.

A comunicação entre frontend e backend ocorre através de requisições HTTP utilizando o padrão REST, permitindo que as páginas realizem operações como login, cadastro e manipulação de produtos consumindo os endpoints da API. Essa arquitetura permitiu desenvolver um sistema simples, organizado e funcional, facilitando futuras melhorias e expansões do projeto.
